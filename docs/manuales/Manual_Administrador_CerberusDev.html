1. Introduccion al Sistema

1.1 Bienvenido Administrador

Bienvenido al Panel de Administracion de Cerberus Dev. Como administrador, tienes acceso completo a todas las funcionalidades del sistema, incluyendo la gestion de usuarios, servicios, contenido y configuracion general.

1.2 Tu Rol como Administrador

El rol de Administrador (Admin) tiene control total sobre el sistema:

  • Gestion completa de usuarios (crear, editar, eliminar)
  • Control total de servicios PM2
  • Administracion de leads y oportunidades
  • Gestion de tickets de todos los clientes
  • Publicacion y edicion de blog y proyectos
  • Monitoreo del servidor en tiempo real
  • Configuracion del sistema
Responsabilidad: Con el acceso total viene una gran responsabilidad. Las acciones administrativas afectan a todos los usuarios del sistema. Actua con precaucion, especialmente al eliminar datos.

1.3 Componentes del Sistema

Modulo Descripcion
Dashboard Centro de control con estadisticas y resumen
Usuarios Gestion de cuentas de usuario y roles
Servicios Control de procesos PM2 del servidor
Monitor Estado del servidor en tiempo real
Leads Contactos y oportunidades de negocio
Tickets Sistema de soporte al cliente
Blog Publicacion de articulos
Proyectos Portafolio de trabajos realizados

5. Gestion de Servicios (CRUD)

5.1 Que son los Servicios?

Los servicios son aplicaciones Node.js ejecutadas mediante PM2 (Process Manager). El sistema permite registrar estos servicios y controlarlos desde el panel web.

5.2 Ver Lista de Servicios

Accede a "Servicios" en el menu para ver todos los servicios.

INSERTAR CAPTURA AQUI
Descripcion: Tabla de servicios registrados
Mostrar: Lista con nombre, estado, acciones de control

5.3 Crear Nuevo Servicio

1 Primero, inicia el proceso con PM2 en el servidor:
pm2 start app.js --name "mi-servicio"
2 En el panel, haz clic en "Nuevo Servicio".
3 Completa el formulario:
  • Nombre: Nombre descriptivo
  • Nombre PM2: Debe coincidir exactamente con el nombre en PM2
  • Descripcion: Detalle del servicio
  • Puerto: Puerto de red (opcional)
  • Cliente: Usuario propietario (opcional)
4 Guarda el servicio.
INSERTAR CAPTURA AQUI
Descripcion: Formulario de creacion de servicio
Mostrar: Campos de nombre, nombre PM2, descripcion, puerto
Importante: El "Nombre PM2" debe ser exactamente igual al nombre que usaste al iniciar el proceso con pm2 start. De lo contrario, las acciones de control no funcionaran.

5.4 Editar Servicio

Puedes modificar los datos de un servicio existente:

1 Haz clic en "Editar" del servicio deseado.
2 Modifica los campos necesarios.
3 Guarda los cambios.

5.5 Eliminar Servicio

Nota: Eliminar un servicio del panel NO detiene el proceso en PM2. Solo elimina el registro del sistema. Si deseas detener el proceso completamente, primero usa "Stop" y luego elimina el registro.

5.6 Controlar Servicios PM2

5.6.1 Iniciar (Start)

Inicia un servicio que esta detenido.

Comando ejecutado: pm2 start [nombre]

5.6.2 Reiniciar (Restart)

Reinicia un servicio. Util despues de actualizaciones o para resolver problemas.

Comando ejecutado: pm2 restart [nombre]

5.6.3 Detener (Stop)

Detiene completamente un servicio. Solo disponible para administradores.

Comando ejecutado: pm2 stop [nombre]
Precaucion: Detener un servicio puede afectar a los clientes que lo utilizan. Asegurate de notificar si es necesario.

5.6.4 Ver Logs

Muestra los registros recientes del servicio para diagnostico.

INSERTAR CAPTURA AQUI
Descripcion: Ventana de logs de servicio
Mostrar: Registros con timestamps y mensajes

Estado Descripcion Accion
online Funcionando correctamente Ninguna necesaria
stopped Detenido Iniciar si es necesario
errored Error en ejecucion Revisar logs y reiniciar

11.3 Middleware de Seguridad

El sistema usa middlewares para proteger las rutas:

  • requireAuth: Verifica que el usuario este autenticado
  • requireRole: Verifica que el usuario tenga el rol necesario
  • rateLimit: Limita intentos de login para prevenir ataques

12. Configuracion del Sistema

12.1 Variables de Entorno

El sistema se configura mediante un archivo .env. Las principales variables son:

Variable Descripcion
PORT Puerto del servidor (default: 3000)
DB_HOST Host de la base de datos
DB_USER Usuario de MySQL
DB_PASS Contrasena de MySQL
DB_NAME Nombre de la base de datos
SESSION_SECRET Secreto para sesiones
ADMIN_PATH Ruta del panel admin
Seguridad: Nunca compartas las variables de entorno. Asegurate de que el archivo .env no este en control de versiones.

12.2 Base de Datos

El sistema utiliza MySQL como base de datos. Las tablas principales son:

  • admin_users: Usuarios del sistema
  • services: Servicios PM2 registrados
  • leads: Contactos y oportunidades
  • tickets: Tickets de soporte
  • ticket_responses: Respuestas a tickets
  • blog_posts: Articulos del blog
  • blog_categories: Categorias del blog
  • blog_comments: Comentarios en articulos
  • projects: Proyectos del portafolio
  • project_technologies: Tecnologias por proyecto
  • project_images: Imagenes de proyectos

12.3 PM2

PM2 es el gestor de procesos utilizado para ejecutar las aplicaciones Node.js. Comandos utiles:

Comando Descripcion
pm2 list Ver todos los procesos
pm2 start app.js --name "nombre" Iniciar proceso
pm2 restart nombre Reiniciar proceso
pm2 stop nombre Detener proceso
pm2 logs nombre Ver logs
pm2 monit Monitor en terminal

12.4 Uploads

Los archivos subidos se guardan en:

  • Proyectos: /public/uploads/projects/
  • Blog: Configurado en el sistema de uploads
Backup: Recuerda hacer copias de seguridad regulares de la carpeta uploads y la base de datos.
